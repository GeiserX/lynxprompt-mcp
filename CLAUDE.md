# CLAUDE.md — LynxPrompt MCP

## Overview
An MCP (Model Context Protocol) server that exposes any LynxPrompt instance to LLMs, enabling them to browse, search, and manage AI configuration blueprints via JSON-RPC.

## Tech Stack
- Go 1.25
- mcp-go (MCP SDK)
- godotenv (config)
- Docker (drumsergio/lynxprompt-mcp)
- GoReleaser (releases)
- npm wrapper (stdio transport via npx)

## Development
```bash
# Run locally
go run ./cmd/server

# Build
go build -o lynxprompt-mcp ./cmd/server

# Run via npm (stdio transport)
npx lynxprompt-mcp

# Docker
docker compose up
```

## Architecture
- `cmd/server/` — main entrypoint
- `internal/resources/` — MCP resource handlers (blueprints, hierarchies, user)
- `internal/tools/` — MCP tool handlers (search, create, update, delete)
- `client/` — LynxPrompt API client
- `config/` — configuration loading
- `.goreleaser.yaml` — multi-platform release config
- `glama.json` — Glama.ai registry metadata
- `server.json` — MCP server descriptor
- `run.js` / `postinstall.js` — npm wrapper for stdio transport
- `Dockerfile` — container build

## Key Rules
- Never hardcode API tokens; use LYNXPROMPT_TOKEN environment variable
- Docker images use semver tags on Docker Hub
- Published on Official MCP Registry, Glama, MCPServers.org, mcp.so, and ToolSDK
- HTTP transport listens on 127.0.0.1:8080 by default (must use reverse proxy for network exposure)
- npm package (`lynxprompt-mcp`) wraps the Go binary for stdio transport
